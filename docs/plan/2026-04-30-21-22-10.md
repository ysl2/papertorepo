# SQL summary source table totals

## Summary

- SQL mode keeps showing the current SQL result row count.
- When SQL column provenance has exactly one supported source table (`papers`, `jobs`, or `exports`), show `x / total rows` to match the normal table summary style.
- When provenance is missing, multi-table, or otherwise unclear, keep `SQL · x rows`.

## Key Changes

- Add `jobs` to the existing dashboard stats response so the frontend has a real full-table total for SQL results sourced from `jobs`.
- In the frontend, derive a unique supported SQL source table from `sqlResult.columnSources`.
- Use dashboard totals for `papers`, `jobs`, and `exports` when the SQL source table is uniquely identifiable.
- Keep detail drawer behavior, SQL execution/cancel behavior, and database schema unchanged.

## Test Plan

- Backend:
  - Add/update dashboard API test coverage for `jobs` total.
  - Run existing public API and SQL endpoint tests.
- Frontend:
  - Run `npm run build`.
  - Manually verify `papers`, `jobs`, and `exports` SQL summaries show `x / total rows`.
  - Manually verify aggregate or unclear provenance SQL still shows `SQL · x rows`.

## Assumptions

- `x / total rows` in SQL mode means `SQL returned rows / unique source table full total`.
- Only `papers`, `jobs`, and `exports` use source table totals for this UI.
