# Fix CI SPA Fallback Test Plan

## Context

GitHub Actions failed in `tests/test_public_api.py::test_frontend_spa_route_still_falls_back_to_index` because CI runs backend tests before building `frontend/dist`. The test currently uses the module-level app, so whether the SPA fallback route exists depends on local build artifacts.

## Plan

- Make the SPA fallback test create a minimal temporary frontend dist with `index.html`.
- Set `FRONTEND_DIST_DIR` to that temporary dist before creating the test app.
- Use `create_app()` instead of the module-level app for the SPA fallback test so route registration is deterministic.
- Reproduce the previous CI failure with a missing default dist override, then run the targeted test and full test suite.

## Scope

- Test-only change.
- No production behavior change.
