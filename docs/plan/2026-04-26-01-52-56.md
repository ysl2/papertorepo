# Frontend Static Asset Fallback Fix Plan

## Goal

Fix the Docker-deployed frontend blank/open failure caused by stale cached asset URLs receiving `index.html` instead of a real asset error.

## Scope

1. Add API app tests proving existing static files are served and missing `/assets/*` files return `404`.
2. Keep SPA fallback for real frontend routes such as `/papers` or `/jobs`.
3. Update frontend fallback logic so paths under `/assets/` or file-like paths with an extension do not fall back to `index.html`.
4. Serve SPA `index.html` with `Cache-Control: no-cache` so future deploys revalidate the entry HTML.
5. Run targeted tests plus `git diff --check`.

## Verification

- `uv run python -m pytest tests/test_public_api.py`
- `git diff --check`
