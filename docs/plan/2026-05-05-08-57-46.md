# Backend/Frontend Monorepo Structure Refactor

## 背景

当前后端 Python 项目文件、迁移、源码和测试位于仓库根目录，前端位于 `frontend/`。为了让项目边界更清晰，本次将仓库整理为 monorepo：`backend/` 是完整 Python 项目根，`frontend/` 是完整 Vite 项目根，repo root 保留编排、配置、文档和 CI。

## 计划

1. 将 `pyproject.toml`、`uv.lock`、`main.py`、`alembic.ini`、`alembic/`、`src/`、`tests/` 移入 `backend/`。
2. 保留根 `README.md` 和根 `.env.example`，删除 `frontend/README.md`，不新增 backend/frontend 单独 README。
3. 更新后端 settings，使从 `backend/` 运行时仍能向上读取 repo root `.env`，并默认找到 repo root 的 `frontend/dist`。
4. 更新 Dockerfile、docker-compose、GitHub Actions、`.gitignore` 和 README 中的路径/命令。
5. 不改包名、CLI 名、API、数据库 schema、环境变量集合或业务逻辑。
6. 验证 backend pytest、CLI 入口、frontend build 和 docker build。
