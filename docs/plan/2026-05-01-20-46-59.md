# Stable detail drawer height

## Summary

- Keep the detail drawer at a stable size while loading content is replaced by canonical paper, job, or export details.
- Use a CSS-only fix: make the drawer panel occupy the available viewport height and keep overflow scrolling inside the panel.
- Do not add per-detail measurement logic or SQL-specific drawer sizing rules.

## Key Changes

- Frontend:
  - Set `.drawer-panel` to a fixed viewport-relative height matching its current max-height.
  - Keep existing loading placeholders and drawer content unchanged except for layout stability.
