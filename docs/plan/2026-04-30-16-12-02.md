# SQL 搜索框改造 + Reset View Bug 修复

## Summary

分两步：先修 Reset view 不清空搜索框的 bug，再将所有 tab 的搜索框从实时 quickFilter 改为按 Enter 触发的双模式搜索。输入内容全部发后端尝试作为 SQL 执行，成功则表格变为"数据库输出投影仪"展示动态列结果，失败则静默回退到 AG Grid quickFilter。不加人为行数限制。

## Implementation Changes

### 1. Reset View Bug
- AgGridSheet 新增可选 prop `onReset?: () => void`，`resetView()` 末尾调用。
- App.tsx 三个 AgGridSheet 实例传入 onReset 清空搜索输入。

### 2. 搜索框改为 Enter 触发
- 拆分为 `tableSearchInput`（实时输入）和 `tableSearchCommitted`（Enter 提交）。
- quickFilterText 绑定 tableSearchCommitted（非 SQL 模式）或空字符串（SQL 模式）。
- placeholder 统一为常量 `SQL_SEARCH_PLACEHOLDER = 'Search or enter SQL...'`。

### 3. 后端 SQL 执行端点

POST /api/v1/sql，请求体 SqlRequest { query: str }，响应体 SqlResponse：
- ok: bool
- has_result_set: bool
- columns: list[str]
- rows: list[dict[str, Any]]
- row_count: int
- message: str | None

执行方式：通过 psycopg3 simple query protocol（prepare=False）支持多语句。手动事务，异常 ROLLBACK 返回 ok=false + 错误信息。HTTP 始终 200。非原生 JSON 类型转字符串。不加行数上限。
返回列名如重复，则追加递增后缀，避免 JSON 行对象和前端动态列覆盖同名列。

### 4. 前端 SQL 模式
- ok + has_result_set：动态列展示结果，rowKey 以 `SQL_ROW_INDEX_KEY = '__row_idx__'` 为基础并避开查询列名冲突，persistenceId 用 `SQL_RESULT_PERSISTENCE_ID = 'papertorepo-sql-result'`，drawer 不响应点击。summary 显示 `SQL · N rows`。
- ok + 无 result_set（写操作）：表格不变，toolbar 展示 message 反馈（SQL_FEEDBACK_MS = 3000 后自动消失），触发当前 tab 数据重新加载。
- 失败：静默回退 quickFilter。

### 5. Tab 切换 / Reset
- 切换 tab / Reset view 时清空 tableSearchInput、tableSearchCommitted、sqlResult。

### 6. 反馈消息 UI
- AgGridSheet 新增 toolbarMessage?: ReactNode slot。

## Test Plan

- Reset view 清空搜索框
- Enter 触发行为
- SQL SELECT 动态列
- SQL 写操作反馈 + 数据刷新
- 多语句整体失败回滚
- 普通文本静默回退 quickFilter
- Tab 切换清空 SQL 状态
- 后端：合法 SELECT、UPDATE、非法 SQL、多语句、非原生类型序列化

## Assumptions

- PostgreSQL + psycopg3，不兼容 SQLite
- 多语句用 simple query protocol
- 无鉴权、无 statement_timeout（V1）
- 不加行数上限
