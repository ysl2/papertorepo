# 修复 CI 依赖下载源

## Summary

GitHub CI 在 `uv sync --dev` 阶段从 `backend/uv.lock` 固化的清华 PyPI mirror URL 下载 `watchfiles==1.1.1`，返回 `403 Forbidden`。本次修复 lockfile 的 Python 包下载源，并让 CI 以 locked 模式同步依赖，避免 CI 依赖国内 mirror 可用性或在流水线里隐式改锁文件。

## Key Changes

- 使用官方 PyPI index 重新生成或刷新 `backend/uv.lock`。
- 将 CI 依赖同步改为 locked install，并显式使用官方 PyPI default index。
- 保持 `backend/pyproject.toml` 依赖声明不变。
- 不修改业务代码。

## Test Plan

- 确认 `backend/uv.lock` 不再包含 `mirrors.tuna.tsinghua.edu.cn`。
- 运行 `cd backend && env -u UV_DEFAULT_INDEX -u UV_INDEX_URL -u UV_INDEX uv sync --dev --locked --default-index https://pypi.org/simple`。
- 运行 `cd backend && uv run python -m pytest`。
- 运行 `npm --prefix frontend run build`。
- 运行 `git diff --check`。

## Assumptions

- CI 失败是依赖下载源问题，不是应用代码或测试问题。
- GitHub Actions runner 可直接访问官方 PyPI。
