# 收口前端 Runtime Config 默认值

## Summary

本次不引入 `VITE_*`。目标是移除前端应用级 runtime config fallback，让 `/api/v1/runtime-config` 成为前端运行配置的唯一来源；保留 `/health` 并行请求，只用于 header 运行状态 chip。

## Key Changes

- 删除 `App.tsx` 里的 `DEFAULT_RUNTIME_CONFIG`。
- 新增显式配置加载状态：只有 `/runtime-config` 成功后，才启动 dashboard/jobs/papers/repos/exports 等依赖配置的请求和轮询。
- `/runtime-config` 失败时设置错误状态，不使用本地默认值静默继续运行。
- SQL/Search UI 和 SQL mode 判断在 runtime config 未加载前按未启用处理，config 成功后按后端返回值恢复。
- 保留 `/health` 请求，并行加载；它只影响 `queue`、`database`、GitHub token 等 header chip，不阻塞 app 数据加载。
- `AgGridSheet` 的 `displayedKeysSyncThrottleMs` 改为必填 prop，删除组件内部 `DEFAULT_DISPLAYED_KEYS_SYNC_THROTTLE_MS`。
- 不修改 `.env.example` 新增字段，不改后端 Settings 默认值，不引入 `VITE_*`，不改 SQL/job 业务逻辑。

## Test Plan

- 前端验证：
  - `npm --prefix frontend run build`
  - 静态检查确认 `DEFAULT_RUNTIME_CONFIG` 和 `DEFAULT_DISPLAYED_KEYS_SYNC_THROTTLE_MS` 已删除。
  - 确认所有 `AgGridSheet` 使用点都显式传入 `displayedKeysSyncThrottleMs`。
- 后端回归：`cd backend && uv run python -m pytest`。
- 全局检查：`git diff --check`。

## Assumptions

- `/runtime-config` 是前端运行配置唯一权威；前端不维护应用级默认配置副本。
- `/health` 信息仍有 UI 价值，因此保留请求和 header chip。
- 后端 `Settings` 默认值和 `.env.example` 示例值继续保留，这是运行配置体系的必要默认来源，不属于前端 fallback。
