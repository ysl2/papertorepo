# Shorten Refresh Progress Meter

## 背景

表格 toolbar 右侧的 refresh papers/loading 进度条当前会占用较多横向空间，尤其在 refresh papers 加载时会压缩搜索栏。用户希望进度条长度接近 `Refresh papers` 文案本身的视觉宽度。

## 计划

1. 只修改 `frontend/src/App.css`，不改 `AgGridSheet` 结构、加载逻辑或数据行为。
2. 将桌面端 `.sheet-grid-status-slot` 从偏宽的弹性宽度收窄到约 `132px`，减少它对搜索框的挤压。
3. 将 `.sheet-loading-meter.toolbar` 的最大宽度同步限制为 `132px`，让进度条本体和状态槽一致。
4. 保留移动端断点里 `.sheet-grid-status-slot { width: 100%; max-width: none; }` 的行为，避免小屏 loading 状态难以阅读。
5. 运行 `npm --prefix frontend run build` 验证前端构建通过。
