# SQL 结果行点击打开现有详情页

## Summary

修复 SQL 查询结果模式下点击行无法打开详情页的问题。普通表格详情逻辑保持不变；SQL 模式新增显式的动态行打开入口，识别出当前 tab 对应实体 ID 后复用现有 drawer 详情链路。识别不到实体 ID 时不打开详情、不报错。

## Key Changes

- 在 `AgGridSheet` 增加可选接口 `onRowOpen?: (row: TData) => void`。
- 未传 `onRowOpen` 时，保持当前 `rowKey + selectedKey + onSelectedKeyChange` 行选择逻辑。
- 传入 `onRowOpen` 时，行点击直接把 `event.data` 交给父组件处理，不改变普通表格行为。
- 在 `App.tsx` 为 SQL 模式提供 `handleSqlRowOpen(row)`。
- Papers tab：只读取 `row.arxiv_id`；值非空则 `setSelectedPaperId(String(value))`。
- Jobs tab：只读取 `row.id`；值非空则 `selectJob(String(value))`。
- Exports tab：只读取 `row.id`；值非空则 `setSelectedExportId(String(value))`。
- 缺少字段、字段为 `null/undefined`、或 trim 后为空字符串：关闭当前 drawer selection 或保持无 selection，不展示错误。
- 打开详情后继续复用现有详情页数据流。
- 不兼容 `SELECT arxiv_id AS id ...` 打开 Paper 详情，不做通用 SQL Row 详情页，不改数据库 schema。

## Test Plan

- 普通表格 Papers/Jobs/Exports 点击行仍打开对应详情。
- SQL Papers tab 执行 `SELECT arxiv_id, title FROM papers LIMIT 1;`，点击行打开 Paper 详情。
- SQL Papers tab 执行 `SELECT title FROM papers LIMIT 1;`，点击行不打开详情。
- SQL Jobs tab 执行 `SELECT id, job_type FROM jobs LIMIT 1;`，点击行打开 Job 详情。
- SQL Jobs tab 执行 `SELECT job_type FROM jobs LIMIT 1;`，点击行不打开详情。
- SQL Exports tab 执行 `SELECT id, file_name FROM exports LIMIT 1;`，点击行打开 Export 详情。
- 聚合查询如 `SELECT COUNT(*) FROM papers;` 点击行不打开详情、不报错。

## Assumptions

- SQL 结果行打开详情按当前 tab 语义判断，不跨 tab 猜测实体类型。
- 识别失败是正常情况，保持安静，不需要 toast 或错误提示。
