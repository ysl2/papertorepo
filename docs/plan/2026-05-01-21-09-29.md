# Unify detail loading title and export cache

## Summary

- Loading drawer title uses a skeleton placeholder instead of showing the selected entity id.
- Export details use a per-id frontend cache, matching the paper detail cache behavior.
- Job details remain uncached because job state is dynamic.

## Key Changes

- Frontend:
  - Render a `DrawerLoadingLine` in the loading drawer `<h3>` title area.
  - Replace `selectedExportDetail` with `exportDetailsById: Record<string, ExportRow>`.
  - Resolve selected export from `exportDetailsById[selectedExportId]` first, then the export list summary.
  - Store fetched export details by id and avoid refetching cached exports.
  - Do not clear the export detail cache when closing the drawer, switching tabs, or receiving new SQL results.

## Test Plan

- Run `npm run build`.
- Run `git diff --check`.
- Manually verify job loading title no longer shows the job id.
- Manually verify reopening the same export reuses cached detail.

## Assumptions

- Export records are stable enough to cache for the current frontend session.
- Job records should continue to reload through the canonical job API on each selection.
