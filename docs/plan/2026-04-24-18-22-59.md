# sync-papers arXiv worker-shared rate limiter implementation plan

## Summary

Move `sync-papers` arXiv request throttling from per-child-job client instances to a shared worker/event-loop limiter. This preserves the current serial batch-child model while making `SYNC_PAPERS_ARXIV_MIN_INTERVAL` apply across child job boundaries in the same worker process.

## Implementation

- Update `docs/design/backend/01-sync-papers.md` to state that arXiv throttling is shared across `sync-papers` child jobs within the same worker/event loop.
- Let `ArxivMetadataClient` accept an optional `RateLimiter`; keep the existing default behavior for standalone callers.
- Add a `sync-papers` pipeline helper that returns a shared arXiv `RateLimiter` keyed by current event loop and min interval.
- Pass that shared limiter when creating the sync-papers arXiv client.

## Verification

- Add tests proving consecutive sync-papers runs reuse the same limiter for the same interval.
- Add tests proving a changed interval gets a distinct limiter.
- Run `uv run python -m pytest`.
- Run `cd frontend && npm run build`.
