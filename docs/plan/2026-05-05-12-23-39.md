# 继续整理运行时可配置常量

## Summary

把本轮确认“用户可感知或部署可调”的常量补到 `.env.example`，并接入现有配置链路。`SYNC_PAPERS_ARXIV_CATCHUP_MAX_AGE_DAYS=90` 不提取到 env，因为这是 arXiv 上游 catchup 行为边界，不是本项目可调策略。

## Key Changes

- 前端 runtime config 新增 `FRONTEND_COPY_FEEDBACK_MS=500` 和 `FRONTEND_LAUNCH_FEEDBACK_MS=3000`。
- 后端 `.env.example` 新增或展示 `PUBLIC_EXPORT_DOWNLOADS=true`、`CORS_ORIGINS=*`、`REFRESH_METADATA_GITHUB_ANONYMOUS_MIN_INTERVAL=60.0`、`REFRESH_METADATA_GITHUB_GRAPHQL_MAX_CONCURRENT=1`。
- 无 GitHub token 时，GitHub REST metadata min interval 使用 anonymous min interval 作为下限。
- refresh-metadata GraphQL semaphore 使用 env 配置的 max concurrent。
- `CORS_ORIGINS` 支持逗号分隔字符串。

## Test Plan

- 后端 Settings / API 测试覆盖默认值、env override、非正数 frontend runtime 值、CORS origins 解析、GitHub anonymous min interval 和 GraphQL max concurrent 覆盖。
- 运行 `cd backend && env -u UV_DEFAULT_INDEX -u UV_INDEX_URL -u UV_INDEX uv run python -m pytest`。
- 运行 `npm --prefix frontend run build`。
- 运行 `git diff --check`。

## Assumptions

- 不把 arXiv catchup 90 天窗口提取到 env，因为它是上游接口约束。
- 不提取 HTTP retry/timeout、API route limit、UI layout 几何值、SQL cancel timeout、rerun handoff timeout。
- 不新增新的配置端点，继续复用 `/api/v1/runtime-config`。
