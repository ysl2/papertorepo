# 前端 Runtime Config 常量整理

## Summary

本次只整理“前端运行时需要、且适合从根 `.env` 管理”的公开常量。数据流保持为：根 `.env` -> 后端 `Settings` -> `/api/v1/runtime-config` -> 前端 state。

## Key Changes

- 在 `.env.example` 新增 `Frontend runtime config` section：
  - `FRONTEND_ACTIVE_DASHBOARD_POLL_MS=1000`
  - `FRONTEND_IDLE_DASHBOARD_POLL_MS=8000`
  - `FRONTEND_ACTIVE_JOBS_POLL_MS=1000`
  - `FRONTEND_PASSIVE_JOBS_POLL_MS=5000`
  - `FRONTEND_TABLE_REFRESH_POLL_MS=20000`
  - `FRONTEND_PAPER_BATCH_SIZE=1000`
  - `FRONTEND_REPO_PREVIEW_LIMIT=10000`
  - `FRONTEND_JOB_PREVIEW_LIMIT=500`
  - `FRONTEND_DISPLAYED_KEYS_SYNC_THROTTLE_MS=200`
- 后端 `Settings` 新增对应字段，使用正整数校验；无 env 时默认值与当前代码一致。
- `/api/v1/runtime-config` 增加公开字段，API 字段去掉 env 前缀。
- 前端移除对应硬编码常量，改用 `runtimeConfig`；加载完成前使用同值 fallback。
- `AgGridSheet` 增加 `displayedKeysSyncThrottleMs` prop，用 `/runtime-config` 下发的 `200ms` 控制 visible rows 主动上报节流。
- 不整理后端底层常量，不引入 `VITE_*`，不改 Docker/env 加载策略，不改 SQL/job 业务逻辑。

## Test Plan

- 后端 API 测试：
  - `/api/v1/runtime-config` 返回新增字段的默认值。
  - 设置 env override 后，`runtime-config` 返回 override 值。
  - 非正数配置应触发 Settings validation，不允许进入前端配置。
- 前端验证：
  - `npm --prefix frontend run build`
  - 确认 `App.tsx` 不再保留这组硬编码运行时常量。
  - 确认 `AgGridSheet` 的 visible row sync 节流来自 prop，默认/fallback 仍为 `200ms`。
- 全量验证：
  - `cd backend && uv run python -m pytest`
  - `git diff --check`

## Assumptions

- 这次只提取前端运行时可调常量；`COPY_FEEDBACK_MS`、tooltip delay、handoff timeout、HTTP retry、SQL cancel timeout 等内部实现常量暂不进入 env。
- `/runtime-config` 是公开浏览器端点，新增字段必须是非敏感信息。
- 前端 fallback 默认值必须和后端 Settings 默认值、`.env.example` 保持一致。
