# sync-papers repair resume observability

## Summary

Expose read-only observability for `sync-papers` repair jobs before they start running. Pending repair children currently have empty `stats_json`, so users cannot tell how much prior work is resumable. Add a serialized repair resume summary derived from the same attempt series and checkpoint table.

This change does not alter job execution, checkpoint reuse, TTL behavior, or persisted `stats_json`.

## Key Changes

- Add `repair_resume_json` to `JobRead`.
- Populate it only for `sync_papers` repair jobs.
- Include previous attempt status/error/stats and checkpoint counts by surface.
- Show the resume summary in job detail and child job cards when current stats are empty or incomplete.

## Test Plan

- Backend serialization returns prior failed attempt stats and checkpoint counts for a pending repair child.
- Non-repair jobs keep `repair_resume_json = null`.
- Frontend build validates the updated API type and display helpers.
