# 提取 Tooltip 显示延迟配置

## Summary

将 tooltip hover 后的显示延迟从组件内不明确的 `DEFAULT_DELAY_MS = 200` 提取为统一前端 runtime config：`FRONTEND_TOOLTIP_SHOW_DELAY_MS=200`。命名明确表达“tooltip show delay”，并通过现有 `.env` -> 后端 `Settings` -> `/api/v1/runtime-config` -> 前端的数据流下发。

## Key Changes

- 在 `.env.example` 的 `Frontend runtime config` section 新增 `FRONTEND_TOOLTIP_SHOW_DELAY_MS=200`。
- 后端 `Settings`、`RuntimeConfigRead` 和 `/api/v1/runtime-config` 新增 `tooltip_show_delay_ms`。
- 前端 `RuntimeConfig` 增加 `tooltip_show_delay_ms`。
- `HoverTooltip` 将 `delayMs` prop 改为更明确的 `showDelayMs`，并将 fallback 常量改名为 `DEFAULT_TOOLTIP_SHOW_DELAY_MS`。
- `App` 将 runtime config 传给直接使用的 tooltip，并通过 `AgGridSheet` 传给表格 header tooltip。

## Test Plan

## Assumptions

- 使用变量名 `FRONTEND_TOOLTIP_SHOW_DELAY_MS`。
- 只配置 tooltip 显示前延迟，不改变 tooltip 位置、样式、offset、viewport gutter 或隐藏行为。
- 继续沿用现有 `/api/v1/runtime-config` 机制，不引入新的 env 加载方案。
