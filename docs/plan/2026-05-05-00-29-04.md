# Visible Rows Sync Throttle

## 背景

`x / total rows` 中的 `x` 表示 AG Grid 当前实际可见行数。当前加载或 refresh 过程中，rows 已经变化时 AG Grid 上报 `visibleKeys` 的时机不够稳定，移动端/窄屏上容易看到 `x` 长时间停在 `0` 或旧值。

## 计划

1. 只修改前端表格组件相关代码，不碰 `.env.example`、不新增 API endpoint、不做项目目录重构。
2. 在 `frontend/src/components/AgGridSheet.tsx` 新增内部常量 `DISPLAYED_KEYS_SYNC_THROTTLE_MS = 200`。
3. 保留现有 `onFirstDataRendered` 和 `onModelUpdated` 同步逻辑。
4. 当 `rows` 或 `quickSearch` 变化时，使用 200ms 节流调度一次 visible keys 同步；如果已有 pending sync，不重复创建。
5. 同步时仍通过 AG Grid API 读取当前 displayed rows，AG Grid 继续作为 visible rows 的唯一权威来源。
6. 对同步结果做 keys 比较，减少无变化时的重复上报；Reset 后补一次强制延迟同步，避免 parent 清空 visibleKeys 后无法恢复。
7. 运行 `npm --prefix frontend run build` 验证前端构建通过。
