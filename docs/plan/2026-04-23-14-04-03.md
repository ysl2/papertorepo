# 全量彻底改名：旧 repo 发现命令与旧元数据刷新命令切换到新命名

## 摘要

- 全量改名 CLI、API、前端、内部函数、配置键、测试、文档和数据库持久化值。
- 不保留旧命令、旧路由、旧 `JobType`、旧配置键的兼容层。
- 对已有数据库做一次性迁移，迁移完成后只保留新名字。

## 关键改动

### 1. 用户面与代码标识

- CLI：`find-repos`、`refresh-metadata`
- API：`/api/v1/jobs/find-repos`、`/api/v1/jobs/refresh-metadata`
- 运行时入口：`run_find_repos`、`run_refresh_metadata`
- `JobType` 的 repo 发现与元数据刷新主类型、batch 类型全部切换为新名字

### 2. 配置

- 旧 repo 发现设置字段和环境变量统一改为 `find_repos_*`
- 所有内部指标、辅助函数、测试命名同步改成新名字

### 3. 数据库迁移

- 新增一次性 Alembic revision
- PostgreSQL：重命名 `jobtype` enum 值，并重算受影响 job 的 `dedupe_key`
- 非 PostgreSQL：更新 `jobs.job_type` 字段值，并重算 `dedupe_key`
- 迁移后数据库中不再保留旧名字

### 4. 前端与测试

- 前端所有请求名、任务类型判断、任务展示与按钮状态切到新值
- 后端和前端测试统一断言新命名
- README、设计文档、示例命令与环境变量说明同步更新

## 测试

- `pytest`
- `cd frontend && npm run build`
- 额外增加迁移测试，验证旧 job_type 与旧 dedupe_key 升级后全部变成新值

## 假设

- 本次不做历史版本兼容
- 但已有数据库需要可升级，因此迁移脚本允许引用旧持久化值，只用于一次性改写历史数据
