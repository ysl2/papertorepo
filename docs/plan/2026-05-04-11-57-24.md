# Search/SQL 显式模式切换

## 需求对齐

为表格搜索框增加显式 `Search | SQL` 模式，而不是自动识别 SQL。

`SQL_SEARCH_MODE=off` 时搜索框永远是普通搜索。`SQL_SEARCH_MODE=read_only/read_write` 时，前端显示模式切换，并记住浏览器上次选择。SQL 模式下永远走后端 SQL endpoint，成功、空结果、状态消息、拦截和数据库错误都按 SQL 语义展示，不再回退普通搜索。

## Plan

1. 配置和 API
   - 保持 `.env.example` 中 `SQL_SEARCH_MODE=off`，并保留允许值说明。
   - `/api/v1/health` 返回 `sql_search_mode: "off" | "read_only" | "read_write"`。

2. 后端 SQL 执行语义
   - 删除当前只读前缀白名单判断，避免用硬编码判断 SQL 类型。
   - `off`：`/api/v1/sql` 拒绝执行。
   - `read_only`：走单语句只读执行路径，由数据库/driver 拒绝写语句、多语句或非法语法，并透传错误 message；成功后 rollback。
   - `read_write`：保留多语句整体执行，成功展示最后一个 result set/status，失败 rollback 并透传数据库错误。
   - `read_write` 下如果存在 `running` 或 `stopping` job，后端拒绝执行并返回明确 message。

3. 前端显式模式
   - `SQL_SEARCH_MODE=off`：隐藏切换器，强制普通 Search，不调用 `/api/v1/sql`。
   - `read_only/read_write`：在搜索框内部左侧显示紧凑 `Search | SQL` segmented control。
   - 模式选择写入 localStorage；无记录默认 `Search`；`off` 时忽略已保存 SQL 选择。
   - Search 模式只设置 AG Grid quick search。
   - SQL 模式只调用 `/api/v1/sql`，不因为 `ok=false` 或数据库报错回退普通搜索。

4. SQL 结果展示
   - 有 result set 且有行：展示动态表格，summary 继续使用现有 source-table totals 逻辑。
   - 有 result set 但 0 行：展示 SQL 空态 `No SQL rows returned.`，summary 仍按现有 SQL summary 逻辑。
   - 无 result set：展示后端/数据库 status message，例如 `UPDATE 1`。
   - 被后端策略拦截或数据库报错：展示返回 message，不显示普通搜索空态。

5. 验证
   - 后端覆盖 `off`、`read_only` SELECT/UPDATE/多语句、`read_write` UPDATE/多语句、running/stopping job 拒绝。
   - 前端通过 TypeScript build；手工验证 Search/SQL 切换、localStorage 持久化、SQL 空态/error/status。
   - 运行 SQL endpoint/public API 测试和完整 pytest。

## 非目标

- 不新增鉴权系统。
- 不做 SQL parser 或前端 SQL 自动识别。
- 不做多结果 tab、streaming、query history。
