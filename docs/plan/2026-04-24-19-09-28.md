# sync-papers repair checkpoint resume

## Summary

Implement request-level checkpoint resume for `sync-papers` repair runs. A failed child job may have already fetched and persisted many arXiv requests before receiving a 429. A repair re-run should reuse those successful request bodies within the same attempt series and continue from the first missing request instead of re-fetching everything from the beginning.

This change does not alter `SYNC_PAPERS_ARXIV_TTL_DAYS`. The TTL continues to control whether fully completed past-day sync windows need a fresh sync for later metadata updates. Partial failed runs do not update TTL completion state.

## Key Changes

- Add a persisted `sync_papers_arxiv_request_checkpoints` table keyed by `attempt_series_key + surface + request_key`.
- Save successful `sync-papers` arXiv responses for `listing_html`, `submitted_day_feed`, `catchup_html`, and `id_list_feed`.
- Use checkpoints only when the current job is a repair attempt in the same attempt series.
- Re-parse reused checkpoint bodies and re-run normal persistence so partially persisted failed attempts can be repaired safely.
- Keep fresh runs, new force runs, and TTL behavior unchanged.

## Implementation Notes

- Pass job execution context from `process_job()` into `run_sync_papers()`.
- Wrap sync-papers arXiv request calls with a helper that checks checkpoint first for repair jobs and stores checkpoint after successful remote responses.
- `id_list_feed` batch checkpoint keys use the existing stable batch hash key so repair re-runs can resume at batch granularity.
- Missing checkpoint body files fall back to remote fetch.
- Add progress stats for checkpoint reuse without counting reused bodies as provider requests.

## Tests

- A failed first run stores successful listing and metadata batch checkpoints before a later metadata 429.
- A repair re-run reuses stored checkpoints and only calls arXiv for missing metadata batches.
- Listing checkpoint reuse still parses IDs and hydrates metadata.
- Fresh runs do not reuse existing checkpoints from previous attempt series.
- TTL skip and completion behavior remain unchanged.
