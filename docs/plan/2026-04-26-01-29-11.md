# GitHub URL Identity and Metadata Convergence Plan

## Goal

Converge GitHub repository identity and metadata around one canonical `github_url` field, and make `refresh-metadata` capable of completing through GraphQL alone when GraphQL succeeds.

## Scope

1. Add an Alembic migration that renames repository URL columns, canonicalizes existing GitHub URLs, deduplicates case-variant repository rows, removes obsolete metadata columns, and adds the agreed common GitHub REST/GraphQL metadata fields.
2. Update SQLAlchemy models and all backend code paths to use `github_url`, `primary_github_url`, and `github_urls_json`.
3. Update GitHub URL normalization so stored canonical URLs always lowercase owner/repo.
4. Update `refresh-metadata` so GraphQL success writes the full agreed metadata set without REST fallback, while GraphQL failures and no-token paths use REST fallback with the same field set.
5. Remove REST conditional request behavior tied to `etag`, `last_modified`, and `checked_at`.
6. Update API schemas/routes, CSV export, CLI, and frontend TypeScript/UI fields to the new names.
7. Update tests first, then implementation, covering migration behavior, normalization, public API shape, CSV output, and GraphQL/REST refresh-metadata behavior.
