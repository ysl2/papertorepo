# sync-papers 与后端常量命名实现计划

## 摘要

按照当前 `docs/design/backend` 的最终设计，把 `sync-arxiv` 全量迁移为 `sync-papers`，同步更新后端、前端、数据库迁移、配置、文档示例和测试；同时调整 `find-repos` 来源顺序与 provider 配置命名。

## 实现步骤

1. 后端领域命名迁移
   - `JobType.sync_arxiv` / `sync_arxiv_batch` 改为 `sync_papers` / `sync_papers_batch`。
   - CLI 命令改为 `sync-papers`，API 路由改为 `/jobs/sync-papers`。
   - `run_sync_arxiv`、`create_sync_arxiv_job`、相关 batch/scope 函数改为 `sync_papers` 命名。
   - 保留 `arxiv_id`、`ArxivMetadataClient`、`RawFetch.provider = "arxiv"` 等 provider 事实命名。

2. 数据模型与迁移
   - `ArxivSyncDay` / `arxiv_sync_days` 改为 `SyncPapersArxivDay` / `sync_papers_arxiv_days`。
   - `ArxivArchiveAppearance` / `arxiv_archive_appearances` 改为 `SyncPapersArxivArchiveAppearance` / `sync_papers_arxiv_archive_appearances`。
   - 删除未使用的 `arxiv_sync_windows` 模型，并增加 Alembic 迁移处理旧表、索引、job_type 和 dedupe_key。

3. 配置与硬编码常量
   - 公共配置改为 `JOB_QUEUE_WORKER_POLL_SECONDS`、`JOB_QUEUE_RUNNING_TIMEOUT_SECONDS`。
   - sync-papers 配置改为 `SYNC_PAPERS_ARXIV_MIN_INTERVAL`、`SYNC_PAPERS_ARXIV_TTL_DAYS`、`SYNC_PAPERS_ARXIV_ID_BATCH_SIZE`、`SYNC_PAPERS_ARXIV_LIST_PAGE_SIZE`。
   - find-repos 配置改为 `FIND_REPOS_LINK_TTL_DAYS`、`FIND_REPOS_HUGGINGFACE_ENABLED`、`FIND_REPOS_ALPHAXIV_ENABLED`、`FIND_REPOS_HUGGINGFACE_MIN_INTERVAL`、`FIND_REPOS_ALPHAXIV_MIN_INTERVAL` 等。
   - refresh-metadata 配置改为 `REFRESH_METADATA_GITHUB_MIN_INTERVAL`、`REFRESH_METADATA_GITHUB_GRAPHQL_BATCH_SIZE`、`REFRESH_METADATA_GITHUB_REST_FALLBACK_MAX_CONCURRENT`。
   - 移除 `ARXIV_TRANSIENT_RETRY_LIMIT`，arXiv 请求统一走公共 HTTP retry 默认值。

4. find-repos 行为调整
   - 查找顺序改为 `Paper.comment`、`Paper.abstract`、AlphaXiv API、AlphaXiv HTML、Hugging Face API。
   - 删除 arXiv abs HTML 抓取路径和 Hugging Face HTML 抓取路径。
   - 更新 metrics、RawFetch、provider 列表和测试断言。

5. 前端、README 与测试
   - 前端任务 key、展示文案、按钮、运行逻辑改为 `sync-papers` / `sync_papers`。
   - README、`.env.example` 和测试命名同步更新。
   - 执行后端测试与前端 build，补齐迁移和行为测试。

## 验证

- `pytest`
- `cd frontend && npm run build`
- `rg` 检查不应残留产品级 `sync-arxiv` / `sync_arxiv` 或旧配置键；provider 事实语义中的 `arxiv` 可以保留。
